Module MonsterBuffTracker

Tracks buffs for all nearby players.

If learning mode is enabled, it also creates a mapping of monster tp move names to localized buff names. The output is saved to a file of the format monsteractionsPlayerName.lua in the data folder for the addon containing this tracker.

Functions

new ()
Default initializer for a new monster buff tracker.

Returns:

    Monster Monster buff tracker
create_file_if_needed ()
Creates a new output file for the player if one does not already exist.
learn ()
Enables learning. When learning is enabled, a mapping of monster tp moves to buffs will be saved to a file of the format monsteractionsPlayerName.lua in the data folder for the addon containing this tracker.
start ()
Starts tracking monster buffs. Note that this function must be called before monster buffs will be recorded. Before this object is disposed of, stop() should be called.
stop ()
Stops tracking monster buffs and disposes of all registered event handlers.
register_event (event_name, event_handler)
Registers a new event handler.

Parameters:

  • event_name MonsterBuffEvent Event name to register a handler for
  • event_handler function Function to be called when the event is triggered
unregister_event (event_name)
Registers a new event handler.

Parameters:

  • event_name MonsterBuffEvent Event name to be unregistered
